📘 DriverProfit — Guía FlutterFlow

Implementación nativa sin custom code · Solo state variables y condicionales

1 · App State Variables

Crear en App State (Settings → App State)

  • userType — String — ""
  • vehicleType — String — ""
  • vehicleLabel — String — ""
  • costPerKm — Double — 2.5
  • minProfitPerHour — Double — 120.0
  • onboardingDone — Boolean — false
  • orderHistory — JSON List — [] (Persist: true)

2 · Estructura de páginas

Crear 5 páginas en FlutterFlow:

  • SplashRouter — decide si mostrar onboarding o app
  • OnboardingDriverType — pantalla 1
  • OnboardingVehicle — pantalla 2
  • OnboardingConfig — pantalla 3
  • MainApp — calculadora + historial + settings (con BottomNav)

3 · SplashRouter

En On Page Load agregar acción condicional:

  • IF appState.onboardingDone == true → Navigate to MainApp
  • ELSE → Navigate to OnboardingDriverType

4 · OnboardingDriverType

Page State variable local:

  • selectedDriver — String — ""

Layout:

  • Column → Text "¿Qué tipo de conductor eres?"
  • 2× GestureDetector o InkWell sobre Container
  • Cada card: On Tap → Update Page State selectedDriver = "delivery" / "rideshare"
  • Usar Conditional Visibility en el ícono de check: visible si selectedDriver == "delivery"
  • Cambiar color del border con Conditional Value: si seleccionado → #4F7FFF, sino → #1A1E2A

Botón CONTINUAR:

  • Enabled condition: selectedDriver != ""
  • On Tap Actions:
    1. Update App State: userType = selectedDriver
    2. Navigate to OnboardingVehicle

5 · OnboardingVehicle

Page State:

  • selectedVehicle — String — ""

Mostrar opciones condicionalmente:

  • Contenedor Delivery Vehicles: Visible si appState.userType == "delivery" — opciones: bicycle, ebike, motorcycle, car
  • Contenedor Rideshare Vehicles: Visible si appState.userType == "rideshare" — opciones: motorcycle, car

Al seleccionar vehículo:

  • Update Page State selectedVehicle = "motorcycle" (etc.)
  • Cambiar borde/color con Conditional Value

Botón CONTINUAR:

  • Enabled: selectedVehicle != ""
  • On Tap → Update App State vehicleType = selectedVehicle → Set costos predeterminados según vehículo (ver tabla abajo) → Navigate to OnboardingConfig

Costos predeterminados por vehículo:

  • ebike → 1.00 / 0.40
  • motorcycle → 1.50 / 0.50
  • car → 3.00 / 0.80

6 · OnboardingConfig

Campos:

  • 3× TextField con tipo Number, con valor inicial desde App State
  • Guardar en Page State local mientras el usuario edita

Botón EMPEZAR:

  • Update App State: onboardingDone = true
  • Navigate to MainApp (reemplazar stack)

7 · MainApp — Calculadora

Page State variables:

  • payment Double
  • km Double
  • minutes Double
  • travelCost Double
  • realProfit Double
  • profitPerHour Double
  • showResult Boolean = false
  • verdictColor String = ""
  • verdictText String = ""

Botón CALCULAR — On Tap Actions:

Usar Custom Action NO disponible → usar Update Page State con fórmulas:

  • realProfit = payment - travelCost
  • profitPerHour = realProfit / (minutes / 60)
  • Condicional para verdictText:
    • IF profitPerHour > 200 → "Excelente pedido — ACEPTAR" + color verde
    • ELSE IF profitPerHour >= 120 → "Pedido aceptable" + color amarillo
    • ELSE → "No conviene — RECHAZAR" + color rojo
  • showResult = true
  • Guardar en orderHistory via Add to List

Resultado — Conditional Visibility:

  • Widget de resultado: visible si showResult == true
  • Color del container: Conditional Value según verdictColor

8 · Historial

  • Usar ListView con appState.orderHistory como fuente
  • Cada item muestra: payment, km, minutes, realProfit, profitPerHour, verdictText
  • Color del indicador: Conditional Value según verdict guardado

9 · Fórmulas en FlutterFlow

Usar Set From Variable → Combine Text / Math Expression:

  • En FF, las fórmulas se configuran campo por campo en el panel de Actions
  • Para multiplicación: usar el campo Math en Update Page State
  • Alternativamente: crear una Custom Function (no custom code de pago) con Dart simple en el editor de funciones gratuito de FF
// Custom Function (gratis en FF)
double calculateProfit(double p, double km, double min, double cpkm, double cpmin) {'{}'}
  double cost = (km * cpkm) + (min * cpmin);
  return p - cost;
{'}'}
